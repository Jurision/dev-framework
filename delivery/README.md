# delivery/

This framework's **own** standard-11 delivery packages — a worked example of
`standards/11-specification-and-delivery.md` applied to real work in this repo (the framework
eating its own dogfood, the way `controls/` runs its own checks).

- `checklists.feature-spec.md` · `checklists.implementation-plan.md` ·
  `checklists.phase-report.md` — delivering the four adoption checklists.
- `adoption-feedback/` — desensitized findings from real-project adoptions, fed back into the
  standards and controls (no project names, identifiers, or production details).

A project's brief lives in its `README`; these reference it rather than copying it.
