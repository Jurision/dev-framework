# 02 — UI design system

🚧 Outline. To be written. Intended scope:

- **Design target as adjective pairs** — "pull toward / pull away" (clean vs. crowded,
  premium vs. discount, calm vs. busy). Every visual change must move toward the target.
- **Tokens, not magic values** — color, typography, spacing, radius, elevation defined once
  and referenced. No ad-hoc hex/px in components.
- **Hard visual rules** — no fake links (`href="#"`), no invented external URLs, restraint
  over decoration, text wraps/clamps cleanly, no useless micro-labels.
- **Accessibility = WCAG 2.2 AA** (contrast, focus order, keyboard, motion, targets) — a
  release criterion, not "a11y" hand-waving.
- **Required reading before a UI task** + a **visual review checklist**; visual changes
  stop at a screenshot/preview and get approval **before push**.
- **Protected design zones** (finalized documents/brand) move verbatim, never restyled
  (`standards/01` §6).

See `checklists/` (visual review) and `standards/00` §5 (looks-done ≠ works).
